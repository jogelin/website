## **\[Core\] Scope Nx Plugin Project Crystal üíõüíõüíõ**

<UrlEmbed url="https://github.com/nrwl/nx/pull/22379?source=post_page-----6d61ed383040--------------------------------" /> 

In my previous article, ‚Äú[‚õî Target Exclusions in Nx Project Crystal](https://medium.com/javascript-in-plain-english/target-exclusions-in-nx-project-crystal-fbabce816b21),‚Äù I introduced alternatives for excluding certain projects from specific plugins.

For instance, you might not want a `test` target in an `e2e` project simply because there‚Äôs a `jest.config.ts` file present.

Now, **an official method is available** for excluding or including plugins for a specific project. Within your `nx.json` file, you can now specify the `exclude` or `include` options:

```json
{
  "plugins": [
    {
      "plugin": "@nx/jest/plugin",
      "options": {
        "targetName": "test",
        "exclude": ["my-lib-e2e/**/*"]
      }
    }
  ]
}
```

## **\[Gradle\] New Gradle Integration**

[![](/blog/images/63a642c1-ff52-429b-8bca-0b6388259ed8.png)](https://nx.dev/nx-api/gradle?source=post_page-----6d61ed383040--------------------------------)

You can now leverage the new Gradle integration with the [`@nx/gradle:init`](https://nx.dev/nx-api/gradle/generators/init) generator. To initialize, simply run:

```bash
nx g @nx/gradle:init
```

This initialization will also add the `@nx/gradle/plugin` to your `nx.json`:

```json
{
  "plugins": [
    {
      "plugin": "@nx/gradle/plugin",
      "options": {
        "testTargetName": "build",
        "classesTargetName": "test",
        "buildTargetName": "classes"
      }
    }
  ]
}
```

This plugin will:

* Integrate your Gradle projects into the Nx Dependency Tree by scanning all Gradle configurations.
    
* Discover your projects by scanning the Gradle configuration matching the pattern `**/build.{gradle.kts,gradle}`, and then assign the targets `build`, `test`, and `classes`.
    

## **\[Cypress\] New target**`open-cypress` is added by the @nx/cypress/plugin

<UrlEmbed url="https://github.com/nrwl/nx/pull/22556?source=post_page-----6d61ed383040--------------------------------" /> 

If you‚Äôre working with Nx Project Crystal and have a Cypress project that includes a `cypress.config.*` file, the `@nx/cypress/plugin`[will automatically add related targets to your project](https://nx.dev/nx-api/cypress/documents/overview#how-nxcypress-infers-tasks).

A new target, `open-cypress`, will now be added to your project configuration:

```json
"open-cypress": {
  "command": "cypress open",
  "options": {
    "cwd": ".",
  },
```

You have the flexibility to use a different name by specifying the new plugin option `openTargetName`:

```json
{
  "plugins": [
    {
      "plugin": "@nx/cypress/plugin",
      "options": {
        "targetName": "e2e",
        "ciTargetName": "e2e-ci",
        "componentTestingTargetName": "component-test",
        "openTargetName": "open-cypress" // <-- NEW
      }
    }
  ]
}
```
